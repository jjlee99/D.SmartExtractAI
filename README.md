# 🚀 D.SmartExtractAI: 이미지 문서 자동 데이터 전환 솔루션 


# 🥇 LG CNS 도입 추진 예정 (Strategic Adoption Highlight)

## 🎯 솔루션 개요 (Overview)

**D.SmartExtractAI**는 DIGITALSHIP에서 개발한 **이미지 문서 자동 데이터 전환 솔루션**입니다. 정책에 따라 문서를 자동으로 식별하여 데이터를 추출하며, 문서 분류, 문자인식(OCR), 그리고 DB 저장을 한 번에 처리합니다.

> "이미지에서 자동으로 자료를 추출. 문서분류, 문자인식, DB 저장을 한번에." 
### 문제 상황 (Problem) 

* 첨부파일로 쌓여있는 이미지 문서를 데이터로 활용하기 어렵습니다.
* 스캔한 이미지 문서에서 자료를 추출하여 DB에 저장하는 과정이 복잡합니다.
* 쌓여있는 이미지 문서 파일의 내용을 검색하기 어렵습니다.
* **→ 첨부문서 및 스캔문서 자동 데이터 변환이 필수적입니다!**

---

## ✨ 핵심 기능 (Key Features)

D.SmartExtractAI는 첨부(스캔) 문서를 DB로 저장하는 전 과정을 자동화합니다

### 1. 문서 자동 분류 (Auto Classification) 
* **목적 문서 식별:** 첨부(스캔) 파일에서 목적 문서를 자동으로 인식하여 추출합니다
* **AI 학습 용이성:** 목표 서식 기반으로 **1장의 서식만으로도 AI 학습**이 가능합니다
* **이미지 표준화:** 원본 이미지 문서를 표준화(전처리) 처리합니다

### 2. 구역 설정 및 데이터 추출 (OCR Extraction) 
* **서식별 구역 지정:** 서식 형태별로 데이터 추출 범위(구역)를 지정합니다
* **자동 OCR:** 문자인식(OCR)을 이용하여 구역별 데이터를 자동으로 추출합니다

### 3. 교정사전 처리 및 구조화 (Data Structuring) 
* **정확성 향상:** 교정사전(공통/조건별)을 적용하여 문자인식 데이터의 **정확성을 향상**시킵니다
* **처리 내용:** 특수문자 제거, 공통/조건별 교정사전 적용, 데이터 타입별 보정
* **데이터 구조화:** 정제된 텍스트를 매핑 정보 기준으로 영역별로 구조화합니다

### 4. 스케줄 관리 및 모니터링 (Monitoring) 
* **일괄 추출:** 작업 스케줄에 따라 파일에서 **일괄로 데이터 추출**이 가능합니다
* **모니터링 제공:** 프로세스 실행 여부, 진행 단계 순서도 모니터링, 로그 분석 기능을 제공합니다

## ✍️ 솔루션에서의 핵심 개발 기여 요약

## 1. 🧠 AI 문서 분류 및 추출 모델 구축 (AI Model Development)

D.SmartExtractAI의 핵심 기능인 '이미지 문서 자동 분류 및 데이터 추출' 모듈을 설계하고 구현하여 솔루션의 자동화 역량을 확보했습니다.

| 성과 목표 | 주요 활동 및 기술 기여 |
| :--- | :--- |
| **자동 분류 모델 개발** | **`Extra_Tree Model` 개발 및 학습:** PyCaret을 활용, OCR 추출 데이터 기반으로 CSV 데이터셋을 구축하고, 가장 효율적인 `Extra_Tree Model`을 개발 및 학습하여 문서 분류의 정확도와 효율성을 확보했습니다. |
| **성능 최적화 기법 도입** | **테이블/헤더 선 검출 강화:** 문서 변별력 향상을 위해 테이블 선 검출을 강화하고 상단 헤더 OCR 결과를 학습에 활용하여 분류 성능을 획기적으로 개선했습니다. |
| **재학습 시스템 구축** | **제목 영역 분리 및 반복 모델 재학습:** OCR 추출 결과의 정확도를 높이기 위해 제목 영역 분리 설정값을 수정하고, 이에 기반한 반복적인 모델 재학습 프로세스를 구축했습니다. |
| **기능 변수화 및 모듈화** | 사용자가 원하는 모델을 선택하고 공통 적용 기능을 파라미터화할 수 있도록 변수화 구조를 검토 및 개발하여 모듈의 유연성을 확보했습니다. |

## 2. 🛡️ 데이터 정제 및 DB 구조화 (Data Refinement & Structuring)

AI 추출 결과의 정확성을 높이고, 대용량 데이터를 안정적으로 관리하기 위한 DB 구조 설계 및 데이터 정제 로직을 완성했습니다.

| 성과 목표 | 주요 활동 및 기술 기여 |
| :--- | :--- |
| **데이터 교정 사전 시스템 구축** | **공통 및 특수 교정 사전 개발:** OCR 오탈자를 수정하기 위한 `TB_DI_COMMON_CRCTN` 및 `TB_DI_BLOCK_CRCTN`을 설계하고 로직을 작성하여 데이터 정확도를 극대화했습니다. |
| **고도화된 DB 적재 로직** | **OCR 데이터 구조화 및 저장:** OCR 대상 영역별 테이블 3개를 생성하고, OCR 텍스트의 좌표 및 행렬 정보를 활용하여 맵 형식의 딕셔너리 데이터를 DB에 효율적으로 적재하는 코드를 작성했습니다. |
| **DB 스키마 안정화** | **테이블 및 PK 재설계:** `OCR 테이블`의 데이터 타입을 변경하고, PK를 `BILD_ID`에서 `DATA_SEQ_NUM`으로 수정하는 등 시스템 안정성을 위한 DB 재설계 작업을 수행했습니다. |
| **데이터 매핑 기능 구현** | OCR 데이터 매핑을 위한 행과 열 정보를 추출하여 JSON OCR 파일에 반영하는 전처리 로직을 개발했습니다. |

## 3. 🌐 시스템 UI/UX 및 글로벌 대응 (Localization & UI/UX Improvement)

주요 관리 시스템(Apache Airflow)의 UI/UX를 개선하고, 솔루션의 글로벌 확장 가능성을 위한 언어팩 및 번역 기능을 개발했습니다.

| 성과 목표 | 주요 활동 및 기술 기여 |
| :--- | :--- |
| **관리 시스템 한글화 (Airflow)** | **전 메뉴 한글화 및 의역:** Airflow의 메인 메뉴, DAG 하위 요소, 버전 표시 등 모든 UI를 한글화했습니다. 단순 직역이 아닌 **의역을 적용**하고 JSON 변수화를 통해 통합 수정 환경을 구축하여 사용성을 획기적으로 개선했습니다 (한글화 90% 완료). |
| **통합 번역 기능 구현** | Google API 기반 **데이터 번역 코드**를 개발하고 관련 테이블을 설계하여, 추출된 텍스트 데이터에 대한 번역 수행 기능을 구현했습니다. |
| **사용자 중심 용어 정리** | **시스템 용어 표준화:** '배치'를 '프로세스', '작업'을 '작업 로그' 등으로 수정하고 메뉴 순서를 재배치하여 사용자 이해도와 관리 편의성을 높였습니다. |
